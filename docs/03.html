LECTURE 03

TensorLy：张量分解

把一个“大权重”拆成几块“小因子”的乘积 —— 用低秩近似压掉模型里的“低秩冗余”

第一讲压“精度位宽”（FP16），第二讲去“结构冗余”（剪整通道）。这一讲压的是第三种冗余——低秩冗余： 一个看似很大的权重矩阵/张量，其“有效信息”往往集中在少数几个方向上。 就像一张高清大图能用少量主成分近似还原，权重也能用几个小因子的乘积来逼近。 tensorly 0.9.0 就是做这件事的工具：把大层换成几个小层，参数和算量大降、结果几乎不变。

🎯 本讲学完你会……

  • 从 SVD / 矩阵低秩近似 出发，理解“为什么大矩阵能用小因子拼回来”；
  • 说清 CP 分解（秩一之和）与 Tucker 分解（核张量 + 因子矩阵 = 高阶 SVD）的区别与适用；
  • 知道怎么把一个 卷积/线性层分解成几个小层 并搭回 nn.Module，以及为何分解后要微调；
  • 会用 TensorLy 的 tucker / parafac 做分解，并理解选秩是精度与压缩的核心旋钮；
  • 把三讲串成一套组合拳：量化 + 剪枝 + 分解如何叠加去打 pangu-weather 的 U/V/W。

§0　上一讲回顾 & 思考题全解

上一讲要点速览
  • 结构化 vs 非结构化：只有结构化（删整通道/核/头）才在通用硬件上真减体积、显存、延迟；非结构化只是置零，形状没变。
  • 牵连问题：删一个输出通道会波及下游输入、BN、残差、拼接；DepGraph 自动把耦合层归成 Group，成组同删、图仍自洽。
  • 流程：定重要性（L1/L2/Taylor）→ 设剪枝率 → 保护输出层 → 迭代剪 + 微调 → 存/导出。剪枝率越高 U/V 越高、W 越险。
Q1 解答：非结构化 90% 置零为何不省、结构化 30% 却真省 矩阵形状 vs 稀疏掩码：非结构化置零后，权重张量的形状一点没变，零只是“数值为 0”，仍占着存储；通用加速器（GPU/国产 DCU）执行的是稠密矩阵乘/卷积，会老老实实把 0×x 也乘一遍、加一遍——MAC 没省、显存没省。要利用稀疏，得有专门的稀疏算子 + 硬件（如特定 Tensor Core 的 2:4 稀疏），本赛题的通用国产卡上一般吃不到。
结构化：删整通道会让权重张量的维度真的变小（如 out_channels 64→44），剩下的还是稠密小矩阵，在任何硬件上都少算少存。所以删 30% 通道≈真减 30%+，而 90% 零散零在稠密硬件上≈白删。
Q2 解答：残差处只剪 F(x) 分支会怎样、DepGraph 如何处理 y = F(x) + x 要求两路通道数严格一致才能逐元素相加。若只把 F(x) 最后一层的输出通道剪掉一部分、不动 shortcut，两路通道数对不上，相加直接形状错误（崩）。
DepGraph 的处理：它在建图时就识别出“这个加法把 F 的输出通道与 shortcut 通道、以及该 block 的输入通道全绑在一起”，于是把它们放进同一个 Group，剪枝时用同一组通道索引一起剪，保持相加合法。若 shortcut 是恒等映射（无卷积可剪），这种依赖往往会约束整组一致剪，否则该处通道不可单独剪。
Q3 解答：同样减半，一次性 vs 迭代式谁的最终精度更好 通常迭代式更好。原因：① 每剪一点就微调，网络能逐步把被删通道的职责重新分配给保留通道，精度边掉边补；② 每一轮在“已适应的新网络”上重新评估重要性，避免一次性按原始排名误删那些“适应后其实重要”的通道。一次性从原始排名一刀切掉一大块，恢复更难。
时间紧的折中：减少迭代步数（如 2~3 步），或一次性剪到中等比例 + 拉长微调，或对敏感模块用迭代、对冗余模块用一次性。
Q4 解答：pangu-weather 差异化剪枝方案 ① 为何不同比例：模块冗余度/敏感度不同。MLP/FFN 通常过参数化、冗余高，能扛更高比例（0.5）；注意力投影/头承载更独特的关系模式、对容量更敏感，比例要低（0.3）以保精度。
② 必进 ignored_layers：最终预测头（输出气象变量/通道由任务定死，不能改输出维度）、patch embedding 输入投影（输入通道由气象变量数决定、改了就接不上数据接口）。
③ 剪完到拿分还要：用正确样例输入 build DepGraph → 迭代剪 + 在 ERA5 上微调/蒸馏 → 验证集逐指标盯 RMSE/ACC → 导出 ONNX → simplify → FP16（敏感算子 block）→ MIGraphX 上卡编译 → 实测体积、峰值显存、单样本时长、各指标 RMSE/ACC，按 Pareto 选点。

§1　先从矩阵说起：低秩近似与 SVD

开场类比：用“几个主成分”还原一张大图 一张 1000×1000 的灰度图有一百万个像素，但如果它“纹理简单”，往往用几十个主成分（SVD 的前几个奇异值）就能还原得肉眼看不出差别——你只需存“几十个列向量 + 几十个行向量”，远比一百万个数少。 神经网络的权重矩阵也常常如此：它的“有效信息”集中在少数几个方向（低秩），于是可以用几个小因子的乘积来逼近。这就是张量分解压缩的根。

SVD：任何矩阵都能拆成“旋转-拉伸-旋转”

任意矩阵 \(M\) 都能做奇异值分解 \(M = U\,\Sigma\,V^{\top}\)，其中 \(\Sigma\) 对角线上的奇异值从大到小排列，代表各方向的“能量”。只保留前 \(r\) 个最大的奇异值，就得到最优的秩-\(r\) 近似：

把全连接层做低秩分解：参数从 \(mn\) 降到 \(r(m+n)\)

一个线性层权重是 \(W\in\mathbb{R}^{m\times n}\)，有 \(mn\) 个参数。若它近似低秩，就可写成 \(W\approx U V\)，其中 \(U\in\mathbb{R}^{m\times r},\,V\in\mathbb{R}^{r\times n}\)：

核心直觉 一个大层换成两个小层串联（先降到 r 维，再升回去）。只要秩 \(r\) 取得比 \(m,n\) 小很多，参数量 \(mn\to r(m+n)\) 就大幅下降，计算量同理。秩 \(r\) 越小越省、但近似越糙——这是贯穿全讲的旋钮。

§2　从矩阵到张量：CP 分解与 Tucker 分解

为什么需要“张量”分解 卷积层的权重不是二维矩阵，而是四维张量 \([\text{输出通道}\times\text{输入通道}\times k_h\times k_w]\)；注意力里也有高维权重。 矩阵的低秩只有“一个秩”，而高维张量有多个模（mode）、每个模都可以低秩。CP 和 Tucker 就是把矩阵 SVD 的思想推广到张量上的两种方式。

CP 分解：把张量写成“一堆秩一张量之和”

CP（CANDECOMP/PARAFAC）把一个张量近似成 \(R\) 个秩一张量之和，每个秩一张量是几个向量的外积：

CP 分解 \[ \mathcal{X} \;\approx\; \sum_{r=1}^{R} \, a_r \circ b_r \circ c_r \] \(R\) 就是 CP 秩；\(a_r,b_r,c_r\) 是各模上的因子向量，\(\circ\) 是外积。

Tucker 分解：一个“核张量” + 每个模一个“因子矩阵”

Tucker 更通用：把张量压成一个小核张量 \(\mathcal{G}\)，再在每个模上乘一个因子矩阵。它正是 SVD 的高阶推广（高阶 SVD / HOSVD），每个模有各自的秩 \((R_1,R_2,R_3)\)：

Tucker 分解 \[ \mathcal{X} \;\approx\; \mathcal{G} \times_1 A \times_2 B \times_3 C \] \(\mathcal{G}\in\mathbb{R}^{R_1\times R_2\times R_3}\) 是核，\(A,B,C\) 是各模的因子矩阵；秩是一个向量 \((R_1,R_2,R_3)\)。
CP vs Tucker，怎么选 CP：只有一个秩 \(R\)，分解最“瘦”、压缩比可以很高，但秩选取与数值稳定性较难，激进时精度更易掉。
Tucker：每个模独立选秩，更灵活、更稳，精度损失通常更小，是 CNN 卷积层压缩里最常用的一种（CP 是 Tucker 在“核张量为超对角”时的特例）。
实践上，卷积层多用 Tucker（对输入/输出通道两个模降秩），并配合 1×1 卷积换通道数——这正是下图展示的做法。

§3　选秩、替换层与微调：把分解真正用起来

三步走 分解压缩在工程上就是：① 选秩 → ② 把原层替换成分解后的几个小层 → ③ 微调回补精度。

① 选秩：压缩与精度的总开关

  • 秩取太大：几乎不压缩；秩取太小：参数省很多但近似误差大、精度崩。
  • 可手工设，也可用 VBMF（变分贝叶斯矩阵分解） 等方法自动估计每个模的合适秩，省去逐层试。
  • 实操常按“保留 X% 能量（奇异值平方和占比）”来反推秩，兼顾压缩与精度。

② 替换层：分解结果要搭回 nn.Module

分解只是得到了几个因子张量，你得把它们组装成等价的小层串联（用因子初始化新层的权重），替换掉原来的大层。TensorLy 负责“算出因子”，PyTorch 负责“搭成层”。

③ 微调：和剪枝一样，分解后几乎必做

分解 ≠ 等价，是近似 低秩分解是有损近似，替换后精度会掉一些。必须用训练/验证数据微调，让新因子层把误差学回来。和上一讲的剪枝完全一样：压缩 + 微调是一套连招。

§4　TensorLy API：分解一个权重张量

两个主力函数 tensorly.decomposition.tucker（Tucker 分解）和 parafac（CP 分解）。输入一个张量 + 目标秩，返回因子；用因子能重建出近似张量，对比误差。
PYTHON · 用 TensorLy 做分解
import tensorly as tl
from tensorly.decomposition import tucker, parafac
import torch

tl.set_backend("pytorch")            # 让 tensorly 直接吃/吐 torch 张量

W = conv.weight.data            # 卷积权重: [out, in, kh, kw]

# —— Tucker：对 out、in 两个模降秩（核保留 kh、kw 不动）——
core, factors = tucker(W, rank=[16, 8, W.shape[2], W.shape[3]])
W_approx = tl.tucker_to_tensor((core, factors))
print("相对误差:", (tl.norm(W - W_approx) / tl.norm(W)).item())

# —— CP：秩 R 的秩一之和 ——
weights, cp_factors = parafac(W, rank=24)
W_cp = tl.cp_to_tensor((weights, cp_factors))
把 Tucker 因子搭成“三个小卷积” 对卷积做 Tucker-2（只降 in/out 两个模）后，等价于：1×1 卷积（把输入通道降到 \(R_2\)）→ 原大小卷积（但通道更少）→ 1×1 卷积（把通道升回 out）。 用分解得到的因子去初始化这三个新卷积的权重，再替换原层即可。tensorly-torch 还提供了 FactorizedConv 等现成的可分解层，能少写很多手工组装代码。

§5　完整示例：卷积层 Tucker 分解，量化收益

目标 取一个卷积层，做 Tucker 分解 → 组装成三个小卷积 → 替换 → 对比参数量、近似误差，并明确“接下来要微调”。
PYTHON · 分解并替换一个卷积层
import torch, torch.nn as nn, tensorly as tl
from tensorly.decomposition import partial_tucker
tl.set_backend("pytorch")

def tucker_decompose_conv(conv, r_in, r_out):
    W = conv.weight.data                       # [out, in, kh, kw]
    # 只对模 0(out)、模 1(in) 降秩
    core, [U_out, U_in] = partial_tucker(W, modes=[0, 1], rank=[r_out, r_in])
    o, i, kh, kw = W.shape
    # ① 1x1：in -> r_in
    first = nn.Conv2d(i, r_in, 1, bias=False)
    # ② 小卷积：r_in -> r_out（保留 kh,kw 与 stride/padding）
    mid = nn.Conv2d(r_in, r_out, (kh, kw), stride=conv.stride,
                    padding=conv.padding, bias=False)
    # ③ 1x1：r_out -> out（带原 bias）
    last = nn.Conv2d(r_out, o, 1, bias=True)
    # 用分解因子初始化三层权重
    first.weight.data = U_in.t().unsqueeze(-1).unsqueeze(-1)
    mid.weight.data = core
    last.weight.data = U_out.unsqueeze(-1).unsqueeze(-1)
    if conv.bias is not None: last.bias.data = conv.bias.data
    return nn.Sequential(first, mid, last)

conv = nn.Conv2d(256, 256, 3, padding=1)
p0 = sum(p.numel() for p in conv.parameters())
block = tucker_decompose_conv(conv, r_in=64, r_out=64)
p1 = sum(p.numel() for p in block.parameters())
print(f"参数量: {p0} -> {p1}  (压到 {p1/p0:.1%})")
# 之后：把 block 替换进模型对应位置，再在数据上微调几个 epoch
你会看到的典型结果（量级感） 一个 256×256×3×3 ≈ 59 万 参数的卷积，降到 r_in=r_out=64 后，参数大致压到原来的三到四成（具体随秩而变）；MACs 同步下降。 刚替换时精度会掉，微调后基本回补。秩越小压得越狠、回补越难——又回到那个核心旋钮。

§6　易错点 · 三讲总收尾 · 赛题落点

⚠️ 易错点汇总

  • 秩选太小：参数省很多但精度崩、微调也救不回——从保守的秩开始、用 VBMF 辅助。
  • 分解完不微调：低秩是近似，不微调精度就一直掉。
  • 所有层都分解：1×1 卷积、很小的层分解收益小甚至变多，挑“大而冗余”的层下手。
  • 因子组装时维度/转置搞错：搭回 nn.Module 时务必核对各因子的形状与卷积权重布局，先用“重建误差”自检再替换。
  • backend 没设：忘了 tl.set_backend("pytorch")，会在 numpy/torch 之间反复转、还可能断开计算图。
🎯 三讲总收尾：三种冗余、三把刀、一套组合拳 ① FP16（第一讲）压“精度位宽”：4 字节→2 字节，体积/显存≈减半，几乎不改结构，最该先做。
② 剪枝（第二讲）去“结构冗余”：删整通道/头，DepGraph 保证成组同删，真减体积/显存/延迟。
③ 张量分解（第三讲）压“低秩冗余”：把大层拆成小因子串联，按秩调压缩比。
三者正交、可叠加，且都遵循同一条规律：压缩换来 U/V，精度（W）靠微调/蒸馏守住，关键是找 Pareto 平衡点。
🔗 赛题落点：在 pangu-weather 上的组合策略
  • 分解哪里：pangu-weather 的注意力 Q/K/V/O 投影、MLP 的大线性层，都是低秩分解的好目标；卷积/patch 处用 Tucker。
  • 推荐管线：剪枝(去结构冗余) → 张量分解(压低秩冗余) → 微调/蒸馏回补 → 导出 ONNX → simplify → FP16(压位宽) → MIGraphX 编译。压缩相乘、精度集中回补一次。
  • 对应分数：剪枝+分解主攻 U（体积/显存）与 V（速度）；FP16 再叠一刀；全程在 ERA5 验证集盯 15 个气象指标的 RMSE/ACC 守 W。
  • 调参思路：把“剪枝率 / 分解秩 / 是否 FP16”当成一组旋钮，扫出“压缩率 × 精度”的 Pareto 前沿，挑 U+V+W 总分最高的点。
